# TAPD 最小需求单元模板

> 创建 TAPD 需求时使用此模板。本模板与 `/newdev` 命令联动，AI 会自动填充各字段。

---

## 需求标题

格式：`[AI研发闭环] <简短功能描述>`

## 需求正文

```markdown
## User Story
作为 <角色>，我希望 <功能>，以便 <价值>

## 最小需求单元
<是/否>。<说明>

## 本次只做
1. ...
2. ...

## 本次不做
- ...
- ...

## 输入
- ...
- ...

## 输出
- ...
- ...

## 验收标准
1. <条件1> → <预期结果>
2. <条件2> → <预期结果>

## AI 执行要求
- 必须使用 AutonomousWorkflowRules 自动研发闭环执行
- 覆盖率 >= 90%
- AI Review >= 95
- 禁止实现「本次不做」中的功能

## 自动化状态字段
- 初始状态：open（任务类型使用 TAPD 标准工作流 open→progressing→done→end）
- 迭代：{DEFAULT_ITERATION}
- 优先级：高
```

## 需求拆分原则

当以下情况发生时，必须拆分为多个需求：

| 场景 | 拆分方式 |
|------|----------|
| 多个 API 接口 | 每个接口一个需求 |
| 前端 + 后端 | 前后端分开 |
| 业务逻辑 + 持久化 | 先逻辑、后持久化 |
| 核心流程 + 附加功能 | 核心流程优先 |

## 示例（参考）

### 拆分前（过大需求）

```
标题：实现用户管理系统
内容：包含注册、登录、信息修改、头像上传、权限管理
```

### 拆分后（最小需求单元）

```
需求1：[AI研发闭环] 实现用户注册接口
需求2：[AI研发闭环] 实现用户登录接口
需求3：[AI研发闭环] 实现用户信息修改接口
需求4：[AI研发闭环] 实现用户头像上传接口
```

## 与 /newdev 命令的关系

`/newdev` 命令会自动：
1. 按此模板生成需求标题和正文
2. 判断是否为最小需求单元
3. 过大的需求自动拆分并询问用户
4. 创建 TAPD 需求后自动启动研发闭环
