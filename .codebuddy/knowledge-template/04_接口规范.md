# 04 接口规范

> **使用说明：** 按实际接口替换以下内容。

## <METHOD> <PATH>

### 功能说明
<功能描述>

### 请求方式
<GET / POST / PUT / DELETE>

### Content-Type
<application/json / multipart/form-data / ...>

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| <参数1> | <string/int/file> | <是/否> | <说明> |
| <参数2> | <string/int/file> | <是/否> | <说明> |

### 成功响应

```json
{
  "code": 0,
  "data": {},
  "message": "success"
}
```

### 失败响应

```json
{
  "code": -1,
  "data": null,
  "message": "错误原因"
}
```

### 错误场景

1. <缺少必填参数> → 返回参数校验失败信息
2. <参数格式不合法> → 返回格式校验失败信息
3. <业务逻辑异常> → 返回明确业务错误
